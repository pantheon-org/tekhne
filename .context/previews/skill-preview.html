Overview

This skill provides deterministic TypeScript guidance for production codebases. It covers tsconfig.json tuning, advanced type constructs, and the patterns needed to maximise type safety without sacrificing developer ergonomics.

When to use this skill

  • Configuring or reviewing tsconfig.json for strict correctness
  • Writing generic utilities, mapped types, or conditional types
  • Debugging inferred vs widened type discrepancies
  • Migrating a JavaScript project to TypeScript incrementally

Compiler Configuration

Start with the strictest reasonable baseline and relax only when necessary:

{
  "compilerOptions": {
    "strict": true,
    "noUncheckedIndexedAccess": true,
    "exactOptionalPropertyTypes": true,
    "moduleResolution": "bundler",
    "target": "ES2022",
    "lib": ["ES2022"]
  }
}

Type Narrowing Patterns

Prefer discriminated unions over boolean flags for variant types:

type Result<T> =
  | { ok: true;  value: T }
  | { ok: false; error: string };

function handle<T>(r: Result<T>) {
  if (r.ok) {
    // r.value is T here — no cast needed
  }
}

Rule: Never use as casts in new code. If you need one, you have a modelling problem.

Utility Types Quick Reference

  • Partial<T> — all properties optional
  • Required<T> — all properties required
  • Pick<T, K> / Omit<T, K> — structural subsets
  • ReturnType<F> — extract function return type
  • Awaited<T> — unwrap Promise depth
