# Scenario 4: Determine When a Context File Is NOT the Right Tool

## Context

You receive the following four requests. For each, determine whether a context file
is the right tool or not.

**Request 1:** "Track this bug in JIRA — the login button is broken on Safari."

**Request 2:** "Add a comment to src/auth/token.ts explaining why we use RS256 not HS256."

**Request 3:** "Document the API design for our new payments service so the team can
reference it for the next 2 years — put it somewhere permanent."

**Request 4:** "I need to jot down some ideas about refactoring the notification service.
Just for now while I think through the approach."

## Your Task

Produce a Markdown file saved to `tool-selection.md` that, for each request:

1. States whether a context file IS or IS NOT the right tool.
2. If NOT, states the correct alternative tool (code comment, issue tracker, README/docs).
3. If IS, states which type (plan, justification, scratch) and why.

## Output Spec

File: `tool-selection.md`
