# Orient at Session Start

## Problem/Feature Description

You are starting a new session to help a developer add a caching layer to their Node.js API. Before writing any code, you need to understand what prior knowledge about this project has already been stored in the knowledge graph.

The developer says: "Let's continue working on the API project. I want to add Redis caching to the user profile endpoints."

## Task

Orient yourself using the graphthulhu knowledge graph before taking any action on the codebase. Demonstrate the session-start orientation workflow and explain what you found (or didn't find).

Do NOT create any files or write any code yet. The goal is exclusively to orient from the knowledge graph.

## Expected Behaviour

- Call `graph_overview` to understand the overall state of the knowledge graph.
- Search for relevant prior context about the project and caching.
- Report a concise summary of what prior knowledge exists (pages found, key findings) or confirm that no relevant prior context was found.
- Do NOT document trivial things (e.g., "started a new session") in the vault.
